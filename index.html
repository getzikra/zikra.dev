Here's what actually happened

No polished origin story. Just a real problem that kept getting worse.

😤

The third time explaining the same thing

We'd start a new Claude session and spend the first 20 minutes re-explaining the database schema, the auth approach, the deployment setup. Every. Single. Time. It wasn't Claude's fault — it just couldn't remember. But it was eating our days.

🤝

The team made it worse

When more people joined, the problem multiplied. Decisions made in one session were invisible to everyone else. Two developers working in parallel would arrive at contradictory solutions and not find out until a late-night code review argument.

🔧

So we built the fix ourselves

We started with a simple webhook and a SQLite database. Then added vector search. Then shared it across machines. Then wired it into our hooks so it ran automatically. Six months later it was saving us hours a week. We called it Zikra.

The core idea

Your agents read from the same memory

Zikra is a small server that sits between your AI tools and a database. Every agent — Claude, Cursor, Gemini, whatever you use — can save and search the same pool of context. Architecture decisions, error patterns, requirements, runbook prompts.

One POST request to save something. Another to find it later. It's not magic — it's just memory that actually persists.

  • Hybrid vector + keyword search — finds things by meaning, not just exact words
  • Hooks fire automatically at session end — no manual saving
  • Token budget on results — never floods your context window

The part we're proudest of

It runs automatically — you don't have to think about it

The hardest part of any memory system is actually using it consistently. We solved this with Claude Code hooks. A Stop hook fires the moment a session ends and logs everything automatically. You never have to type "save this" — it just happens.

The statusline shows your memory count and session stats right in your terminal. After a while you stop noticing it's there — you just notice that Claude always seems to know your codebase.

  • Stop hook logs every Claude Code session on close
  • PreCompact hook saves context before the window resets
  • Neovim and Claude Code statusline — live memory stats
