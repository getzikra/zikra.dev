What's Changed

Every release. Every fix. Every improvement.

v1.0.4 2026-04-09

  • MCP transport migrated to Streamable HTTP — the /mcp endpoint now speaks the MCP 2025-03-26 Streamable HTTP spec (stateless POST, JSON-RPC 2.0). Container restarts and proxy reconnects no longer drop sessions. The deprecated SSE endpoint at /mcp/sse is kept for backward compatibility with a logged warning.
  • Safe update script (update.sh) — one command to pull, detect Docker vs. bare-Python, and restart. Checks for uncommitted changes before pulling, snapshots a WIP branch if any exist, then restores after the update. Docker: detects bind-mount or compose and restarts accordingly.
  • Server update prompt (prompts/zikra-server-update.md) — Claude Code prompt that walks through the same update steps for users who prefer not to run scripts directly.
  • Onboarding prompt renamed — prompts/g_zikra.md → prompts/zikra-cli-install-update.md to better describe what it does (configures Claude Code client hooks only). A disclaimer at the top clarifies the scope: hooks, statusline, MCP token — not server updates.
  • SQLite concurrency warning — installer now warns when SQLite is selected with a non-localhost bind address. Single-writer limitation documented in README profiles table.
  • SQLite runtime files untracked — *.db-shm and *.db-wal added to .gitignore. WAL-mode journal files were being accidentally staged on some systems.

v1.0.3 2026-04-09

  • Version badge direction fixed — the topbar update indicator now only shows v→ vX.X.X when GitHub has a strictly newer release. Previously showed a downward arrow when running ahead of the latest tag.
  • Favicon — browser tab now shows the Zikra Z icon via an inline SVG link. No external file required.
  • Memory titles no longer cut off — sidebar card titles use the full available width before truncating with …. A CSS property mismatch (flex:1 on a non-flex child) was causing mid-word clipping.
  • Access counter now increments — get_memory calls bump_access_count on every read. Search results also expose access_count so the accessed N× label in the sidebar is accurate.
  • Memory decay gauge fixed — the circular health gauge and recency/frequency/confidence bars now compute correctly. Root cause: search results were not returning access_count or confidence_score, and the UI was reading the wrong field. Gauge now reads the server-computed score (0–1); bars are derived independently from created_at, access_count, and confidence_score.
  • Added compute_score() to scoring module — standalone memory health score (decay × frequency × confidence × pin boost), independent of search query. Applied to all search results so the UI always has an accurate health signal.
  • Version file synced — version.py bumped to 1.0.3 and git tag v1.0.3 pushed so the GitHub Tags API reflects the current release.

v1.0.2 2026-04-09

  • Single version source of truth — version.py now drives everything: FastAPI, /health, debug_protocol, and the version command. No more three different hardcoded strings.
  • Server-side GitHub version check — /health and the version command return latest_version when a newer GitHub tag exists. One daily API call, cached in memory, shared by all endpoints.
  • Web UI version indicator — the topbar now shows the running server version on connect. Turns orange with v1.0.1 → v1.1.0 if GitHub has a newer release.
  • Stop hook race condition fixed — lock moved to main shell (before the background fork). Was inside the subshell, causing 30-80× duplicate diaries per session.
  • Stop hook: macOS portability — portable lock: flock on Linux/WSL, atomic mkdir fallback on macOS. Hook now works correctly on all platforms.
  • Stop hook: diary title includes time — title is now diary:YYYY-MM-DD-HHMM:host instead of diary:YYYY-MM-DD:host. Multiple sessions per day are preserved instead of being silently overwritten.
  • Stop hook: transcript byte cap — adaptive tail (80–200 lines) + 200KB head -c cap prevents megabytes of JSONL tool results being fed to claude -p.
  • Stop hook: claude -p timeout — timeout 120 (or gtimeout on macOS) prevents a hung call from holding the lock indefinitely.
  • Removed zikra-lite — all references to the discontinued zikra-lite product removed from installer, hooks, docs, and website. One product, one repo.
  • Installer fix — g_zikra.md no longer has a localhost → zikra-lite branch that silently pulled stale hooks from the old repo.

v1.0.1 2026-04-08

  • Statusline: server version display — the Claude Code statusline now shows the running server version fetched from the Zikra API, not a hardcoded local string. Compares against GitHub tags and highlights in orange if an update is available.
  • New version command — returns {"version": "1.0.1", "server": "zikra"}. Accessible to any viewer role token. Aliases: ver, server_version.
  • Stats hook: server version cache — zikra-stats-update.sh fetches the server version each run and caches it in ~/.claude/cache/zikra-stats.json.
  • Statusline fallback — shows (update if ●) with an orange dot when server version is not yet cached, prompting the user to run zikra update.

v1.0.0 2026-04-01

  • Initial release — FastAPI + SQLite + sqlite-vec. Zero external dependencies.
  • MCP server — SSE transport at /mcp, compatible with Claude Desktop and Claude Code.
  • Hybrid search — semantic (vector cosine) + keyword (FTS5), weighted scoring with confidence decay.
  • Claude Code hooks — Stop hook auto-logs session diaries; PreCompact hook saves summaries before context compression; statusline shows live stats.
  • Multi-user RBAC — owner, admin, developer, requirements_engineer, viewer roles. Token-based auth on all endpoints.
  • Memory types — decision, error, requirement, reference, user, feedback, project, conversation, architecture.
  • One-command installer — g_zikra.md prompt sets up hooks, CLAUDE.md, and statusline in a single Claude Code session.
  • PostgreSQL backend — optional DB_BACKEND=postgres for teams needing pgvector scale.

Always improving.

Watch the repo for new releases. Every update ships through the same one-command installer.
